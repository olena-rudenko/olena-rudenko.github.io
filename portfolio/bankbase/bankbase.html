App "Bankbase" by Olena Rudenko


Design exercise:

Banking users struggle to keep track of their overall spending and are not effective at saving money. Design an experience to help a banking user understand and manage their expenses, considering factors including totals, possible expense categories and associated transactions. Help the banking user set achievable saving goals, considering completion periods and notifications.

Process:

We need to know why we need to create this experience. Ideally the process should start by studying the current version and analyzing the services of competitors. We don't have the current version, so we just need to talk to our user by checking user's feedbacks in AppStore and Google play.



Identified deficiencies:

• Each screen contains too much information, which must be scrolled.
• Too many functions on one screen, what complicates the use of the application, so "Less is More".
• Lack of visualization of users spending in a way that is convenient for each user persone individually.
